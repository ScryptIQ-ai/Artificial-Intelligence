    Assignment AI1

    Assignment Background:

    As testers you don't have much time, so please feel free to just attempt what you can. A difficult part is gauging the difficulty of an assignment to not bee too easy, too difficult, or too long.

    This assignments aim is to get you actively creating and experimenting with the architecture of a multi layer perceptron (MLP).

    For this assignment, we will be using the MNIST handwritten digits dataset. The dataset has been split into two parts:

    • Training set (60,000 images): Used to train the model by updating weights

    • Test set (10,000 images): Used to evaluate final model performance

    Question 1

    Create Three Different MLP Architectures

    The MNIST data has already been loaded and split for you (see the cells below).

    Your task is to complete the three MLP class definitions (MLP_1, MLP_2, and MLP_3) provided in the code cells.
    Use the example_MLP class given in the cell below as a guide for the structure.

    Each of your three MLPs should explore a different architectural variation:

    MLP_1 - Deeper Network:

    • Add more hidden layers (aim for 4-5 hidden layers total).

    • Keep the same activation function (ReLU) and hidden layer size (512 neurons) as the example.
    MLP_2 - Different Activation:
    • Keep the same depth as the example (2 hidden layers).

    • Use Tanh activation instead of ReLU throughout the hidden layers.
    MLP_3 - Wider Network:
    • Keep the same depth (2 hidden layers) and activation function (ReLU) as the example.

    • Increase the number of neurons per hidden layer to 1000 instead of 512.
    Available activation functions:
    • nn.ReLU() - Documentation

    • nn.Tanh() - Documentation

    • nn.Sigmoid() - Documentation
    > NOTE: Always keep nn.Softmax(dim=1) as the final activation (this is already given in the starter code).

    What to modify:
    • In __init__: Add or keep nn.Linear() layers as needed

    • In forward(): Add or keep the corresponding activation functions between linear layers

    • The hidden layer size will be set when you instantiate the models in Question 2

    Question 2

    Train and Evaluate Your Models

    Part A: Complete the Model Dictionary

    In the code cell below create a dictionary called model_dict where each model is instantiated and assigned a descriptive name.

    • Ensure each model is initialised with the same input size and output size
    All models should have input_size=784 (flattened 28×28 images) and output_size=10 (10 digit classes).
    Part B: Training and Testing Functions

    The train() and test() functions given in the lesson materials are provided below.

    Review these functions to understand:
    • train(): Updates model weights using training data

    • test(): Evaluates model performance on test data without updating weights
    Your task: Rather than simply printing the result, edit the test() function to return these three values, in this order:
    • correct

    • accuracy

    • test_loss
    Part C: Train All Models

    Your task: Complete the training loop in the code cell below.

    The loop should do the following:
    1. Iterate through each model in model_dict

    2. For each model:
      • Create an SGD optimiser with lr=0.01 and momentum=0.5

      • Train for 6 epochs, calling train() in each epoch

      • After training completes, call test() on the test dataset

      • Store the test results in a dictionary called results
    The results dictionary should have this structure:

    results[model_name] = {
    'Accuracy': accuracy,
    'N. correct (10,000)': correct,
    'Test Loss': test_loss
}


    After the training loop, print the results for each model in a clear format.

    Note: We use the following separate datasets:
    • Training set (60,000 images): Used to update model weights during training

    • Test set (10,000 images): Used to monitor progress during training (optional)
