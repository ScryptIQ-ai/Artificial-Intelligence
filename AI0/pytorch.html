    Introduction

    PyTorch is an open-source machine learning and deep learning framework that provides tools for building, training and deploying neural networks using Python. It is built around the concept of tensors and automatic differentiation, allowing complex mathematical models to be expressed in a clear and flexible way. PyTorch supports both CPU and GPU acceleration and is designed to scale from small experiments on a laptop to large models running on specialised hardware.

    PyTorch is particularly popular in research and education because of its intuitive Pythonic design and its emphasis on dynamic computation graphs, which allow models to be defined and modified on the fly. This makes it intuitive to debug, experiment and iterate compared to more rigid frameworks. At the same time, PyTorch is widely used in industry, powering applications in computer vision, natural language processing, speech recognition and even generative AI. Its balance of flexibility, performance and a large supporting ecosystem has made it one of the most widely adopted deep learning frameworks in use today; many of the attributes that have made Python attractive as a mainstream programming language.

    PyTorch installation

    To install PyTorch, the approach varies depending on your operating system, whether your computer has a GPU and the type of GPU. It is essential to visit the PyTorch installation guide website at PyTorch Get Started Locally for specific instructions tailored to your setup.

    Below, I'll provide general guidance and examples for installing PyTorch on Windows, Linux and macOS systems that have an NVIDIA GPU, using both pip and conda.

    NOTE

    We have provided GitHub CodeSpaces for you with a CPU version of PyTorch installed already. The following is for local setup only.

    Installation prerequisites

    Before installation, make sure you check the following:

    1. GPU compatibility (optional but recommended)
    If your system includes a GPU and you plan to use hardware acceleration, you must firstly check that it is supported.

    • NVIDIA GPUs must be CUDA-capable and compatible with the version of CUDA supported by your chosen PyTorch build. If a GPU is CUDA-capable means that a graphics card can run software built using CUDA (Compute Unified Device Architecture), NVIDIA’s framework for general-purpose computation on GPUs. In practical terms, a CUDA-capable GPU can perform large numbers of mathematical operations in parallel, which is exactly what libraries like PyTorch use to accelerate machine learning workloads.

    • Apple macintosh computers with Apple silicon (such as the M1, M2 and M3 processors) use MPS (Metal Performance Shaders) for GPU acceleration instead of CUDA. This allows PyTorch to use the high-performance GPU built into your Mac for machine learning tasks.

    • AMD GPUs may be supported through alternative backends, but support is more limited and platform-dependent. If you are unsure, PyTorch will still run correctly on the CPU, albeit more slowly for large models.

    2. Python installation

    Ensure that Python is installed and accessible from the command line. PyTorch supports Python version 3.7 and up. You can check your installed version by running python --version or python3 --version in the terminal. If multiple Python versions are installed, make sure you use a compatible one.

    3. Environment setup

    It is strongly recommended to install PyTorch inside a virtual environment rather than into your system-wide Python installation. Virtual environments help isolate project dependencies, reduce version conflicts between packages, and make your work easier to reproduce. Common options include venv, Conda environments, or other environment managers you may already be using. Please refer to our Python Orientation reference materials for more information on how to set up a virtual environment.

    Taking a few minutes to verify these points will help avoid common installation issues and make working with PyTorch more reliable from the outset.

    FACT

    PyTorch tensors and models live on a device, typically either the CPU or a GPU. While code can often be written in a device-agnostic way, it is common to explicitly move data and models between devices for performance reasons. In machine learning workloads that involve large matrix and tensor operations, GPUs can provide immense advantage, often being orders of magnitude faster than CPUs on a standard desktop or laptop; this often makes them far better suited to training and running modern AI models.

    Managing your devices correctly is essential for avoiding common runtime errors and for taking advantage of hardware acceleration.

    Recap: creating a virtual environment

    Before we dive into installing PyTorch, let's recap on what a virtual environment is.

    KEY TERMS

    A virtual environment is an isolated Python environment that allows you to install packages for a specific project without affecting your system-wide Python installation or other projects. Using virtual environments helps to compartmentalise your work, and avoid version conflicts, making projects easier to reproduce, and is considered a best practice for working with Python libraries such as PyTorch.

    Why use a virtual environment?

    Virtual environments allow you to:

    • Keep project dependencies separate

    • Avoid conflicts between package versions

    • Ensure consistent behaviour across machines

    • Safely experiment without breaking other projects
    Each virtual environment has its own Python interpreter and its own set of installed packages.

    REMEMBER

    Python includes a built-in tool called venv for creating virtual environments.

    In your terminal, run the following line from the root of your working directory:

    python -m venv .venv

    Choosing between pip, UV, and conda

    Both tools are valid for installing PyTorch. The best choice depends on how you manage your Python environments.

    • Use pip if you are already working with venv and standard Python packaging workflows

    • Use conda if you are managing environments and scientific dependencies through Anaconda or Miniconda
    Whichever option you choose, keep it consistent within one project to reduce dependency conflicts.

    WARNING

    Do not mix installs in the same environment unless you have a clear reason and understand the dependency implications.

    CPU-only installation

    If you do not have a compatible GPU, or if you are just getting started, you can install and run the CPU only version of PyTorch. This is often enough for early learning and smaller experiments.

    In your terminal, use one of the following:

    Using pip:

    pip3 install torch torchvision torchaudio


    Using conda:
    conda install pytorch torchvision torchaudio cpuonly -c pytorch

    Installing PyTorch (NVIDIA)

    As referenced above, systems with an NVIDIA GPU will make use of the CUDA toolkit to leverage GPU acceleration. Here are installation examples for pip and conda on different operating systems.

    In your terminal (using shell), type the following:

    Using pip

    pip3 install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121


    Using conda

    conda install pytorch torchvision torchaudio pytorch-cuda=12.1 -c pytorch -c nvidia

    Installing PyTorch (Apple Silicon)

    For Apple Macintosh computers with Apple Silicon (M1, M2, M3 chips) use MPS (Metal Performance Shaders) for GPU acceleration instead of CUDA. The first step to take before installing, is to determine which version of Mac OS your computer is running. You can do this by clicking the Apple logo in the top-left corner of your screen and selecting "About This Mac".

    NOTE

    To use PyTorch on Apple silicon, you will need to be running Mac OS 12.3 (Monterey), or later. Previous versions do not include the MPS support essential to running PyTorch.

    It is highly recommended that you use a native arm64 version of Python, rather than one installed via Rosetta 2. You can install these via Miniforge or the official installers available at Python.org.

    Using pip

    Enter the following into your terminal app (bash):

    pip3 install torch torchvision torchaudio

    Using conda

    Enter the following into your terminal app (bash):

    conda install pytorch::pytorch torchvision torchaudio -c pytorch

    Installing PyTorch (Windows & Linux)

    For computers running Windows or Linux, GPU acceleration is typically provided via CUDA for systems with NVIDIA GPUs, or via AMD’s equivalent GPU compute platforms on supported hardware.

    To use GPU acceleration with PyTorch, your system must have a compatible GPU and the appropriate drivers installed. Before installing PyTorch, the first step is to determine whether your machine has an NVIDIA or AMD GPU and to check your operating system version.

    • On Windows, you can do this via Settings → System → About or through Device Manager.

    • On Linux, you can check your system details using commands such as lspci (to list hardware) and uname -a (to view OS information). If no compatible GPU is available, PyTorch can still be installed and used with CPU support.

    CUDA installation commands

    Using pip

    Into the terminal (shell), enter the following line:

    pip3 install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121

    Using conda

    Into the terminal (shell), enter the following command:

    conda install pytorch torchvision torchaudio pytorch-cuda=12.1 -c pytorch -c nvidia

    Importing PyTorch

    Once installed, as with any Python package, PyTorch needs to be imported. We can also simultaneously implement import verification, as follows:

    NOTE

    The first check confirms that PyTorch is installed and importable in your active environment. If this fails, double-check that your notebook kernel is using the same environment where you installed PyTorch.

    import torch

print(torch.__version__)
    2.5.1+cpu

    For our setup we have chosen the 2.5.1 version, as this is the most up to date, CPU only version. As of writing the latest stable version was 2.9.1.

    Finding out what is avaiable

    The following code is used in the lessons to check whether there is a GPU or MPS available, if not it falls back to the CPU.

    import torch
import torch.nn as nn

# Check if GPU is available
if torch.cuda.is_available():
    # if there are multiple GPUs, choose the first one
    device = torch.device("cuda") # Defines the device you want to have data moved to
    print(f'There are {torch.cuda.device_count()} GPU(s) available.')
    print(f"Using: {device} (Cuda)")

# Check for MPS (Apple Silicon GPU) availability
elif torch.backends.mps.is_available():
    device = torch.device("mps")
    print(f"Using: {device} (Apple Silicon)")

else:
    print("No GPU detected! Falling back to CPU")
    # If the GPU is not available, use the CPU
    device = torch.device("cpu")

# Create a simple model
model = nn.Linear(10, 2)

# Move the model onto the chosen device
model.to(device)

    Common installation issues

    IMPORTANT

    If you run into setup problems, check the following:

    • PyTorch was installed into the same environment used by your notebook kernel

    • Your Python version is supported by the installed PyTorch build

    • GPU drivers and backend support (CUDA or MPS) are installed and compatible

    • You restarted the kernel after installation so imports reflect the updated environment
